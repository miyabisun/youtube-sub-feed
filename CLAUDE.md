# youtube-sub-feed プロジェクトルール

## ドキュメント

- `docs/*.md` はコード変更時に常に最新化すること

## 開発

- `/dev` で実装する際は TDD (Red → Green → Refactor) で進めること
- `client/` 配下の変更後は必ず `cd client && bunx vite build` で成功確認

## レビュー

- `/rev` でのレビュー時、テストに関しては **「お前それt-wadaの前で同じこと言えんの？」** の視点でチェックすること
  - テストが書かれていない新規ロジックはないか
  - テスト名は振る舞いを説明しているか（実装詳細に依存していないか）
  - エッジケース・異常系・境界値が考慮されているか
  - テストが実装と一緒に書かれているか（後付けになっていないか）
